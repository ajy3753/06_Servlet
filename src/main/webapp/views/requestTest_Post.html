POST 방식으로 요청해보기

특징 1. POST 방식으로 요청시 URL의 BODY 영역에 데이터를 포함시켜 넘김
=> 사용자가 입력한 값들이 URL에 노출되지 않음
=> 따라서 보안유지가 가능
=> 즉, 로그인이나 회원가입 같은 경우 POST 방식이 적합

특징 2. BODY 영역은 전송하는 길이에 제한이 없음
=> 즉, 게시판 작성 같은 경우 POST 방식이 적합

특징 3. 즐겨찾기는 가능하나 데이터가 URL에 노출되지 않음
=> URL을 저장했다가 다시 요청해도 기존에 봤던 응답화면을 볼 수 없음
=> 즉, 검색이나 북마크 기능 같은 경우 POST 방식에 부적합

특징 4. Time out이 존재하기 때문에 페이지 요청, 기다리는 시간 존재
